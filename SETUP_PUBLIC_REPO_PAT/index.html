#Setting Up PUBLIC_REPO_PAT Secret

The build-and-sync-public.yml workflow needs a Personal Access Token (PAT) to push to the public repository on github.com.

#Why a PAT is Needed

  • The internal repo is on github.ibm.com (GitHub Enterprise Server)
  • The public repo is on github.com (public GitHub)
  • Cross-platform authentication requires a PAT from github.com

#Steps to Create and Configure

#1. Create a Personal Access Token on github.com

  1. Go to https://github.com/settings/tokens
  2. Click "Generate new token" → "Generate new token (classic)"
  3. Configure the token:
    • Note: ZDEP Docs Sync Token
    • Expiration: Choose appropriate expiration (90 days, 1 year, or no expiration)
    • Scopes: Select:
      • ✅ repo (Full control of private repositories)
        • This includes repo:status, repo_deployment, public_repo, repo:invite, security_events
  4. Click "Generate token"
  5. IMPORTANT: Copy the token immediately - you won't be able to see it again!

#2. Add Token as Repository Secret

  1. Go to the internal repository: https://github.ibm.com/zEco-IaaS/zdep-docs
  2. Navigate to Settings → Secrets and variables → Actions
  3. Click "New repository secret"
  4. Configure the secret:
    • Name: PUBLIC_REPO_PAT
    • Value: Paste the token you copied from github.com
  5. Click "Add secret"

#3. Verify Token Permissions

The token must have access to the public repository:

  1. Go to https://github.com/IBM-WSC/zdep-docs
  2. Ensure the user who created the PAT has write access to this repository
  3. If not, add them as a collaborator with write permissions

#Testing the Setup

After adding the secret, trigger the workflow:

# Make a small change and push
cd beamz-zos-demo-docs
echo "# Test" >> README.md
./build-both.sh
git add -A
git commit -m "Test public repo sync"
git push origin main

Check the workflow run at: https://github.ibm.com/zEco-IaaS/zdep-docs/actions

If successful, you should see:

  • ✅ Internal site deployed
  • ✅ Files synced to public repo
  • ✅ Public site deployed

#Troubleshooting

#"Bad credentials" Error

  • The PAT is invalid or expired
  • The PAT doesn't have repo scope
  • The secret name is wrong (must be exactly PUBLIC_REPO_PAT)

#"Resource not accessible by integration" Error

  • The user who created the PAT doesn't have write access to IBM-WSC/zdep-docs
  • Add them as a collaborator with write permissions

#Token Expiration

When the token expires:

  1. Create a new token following the same steps
  2. Update the PUBLIC_REPO_PAT secret with the new token value

#Security Best Practices

  • ✅ Use a token with minimal required scopes (repo only)
  • ✅ Set an expiration date (90 days or 1 year recommended)
  • ✅ Rotate tokens regularly
  • ✅ Never commit tokens to the repository
  • ✅ Use a dedicated service account if available

#Alternative: GitHub App

For better security and no expiration, consider using a GitHub App instead of a PAT:

  • More granular permissions
  • No expiration issues
  • Better audit trail
  • Requires more setup

See: https://docs.github.com/en/apps/creating-github-apps/about-creating-github-apps/about-creating-github-apps
