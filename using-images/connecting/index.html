#Connecting to Your Instance in TechZone

Learn how to connect to a provisioned z/OS demo instance from TechZone.

#Prerequisites

  • A provisioned z/OS instance from TechZone in Ready state
  • z/OS IP address (from the TechZone reservation details)
  • 3270 terminal emulator installed (e.g., IBM Host On-Demand)

#Step 1: Connect to VPN

Network access to the z/OS instance is required before connecting. How you connect depends on whether you are an IBMer or a Business Partner.

#IBMers

Connect to the IBM network using Cisco AnyConnect as you normally would. No additional VPN is required.

#Business Partners / External Users

VPN credentials are provided with your TechZone reservation. Use one of the following methods:

Option A: Cisco AnyConnect

  1. Install Cisco AnyConnect if not already installed
  2. Open Cisco AnyConnect
  3. Enter the VPN server address from your TechZone reservation details
  4. Enter the provided VPN credentials
  5. Click Connect

Option B: Jump Server

If it's not possible or too inconvenient to install Cisco AnyConnect on your workstation, provision and use the browser-based Windows desktop jump server here.

#Step 2: Set Your TSO Password via SSH

If TSO access is needed, before connecting via 3270 terminal, the TSO password must be reset by first connecting via SSH key and resetting the password manually:

#2.1 Download and Prepare Your SSH Key

Your TechZone reservation provides an SSH key for authentication. Rename it to something simple like zdep-key.pem before proceeding — the default filename contains spaces and parentheses that will cause SSH to fail.

If this is your first time downloading the key:

mv ~/Downloads/pem_user_privatekey_download.pem ~/Downloads/zdep-key.pem

⚠️ If you have downloaded a TechZone key before, your browser may have saved it with a number appended. It must be renamed using quotes: mv ~/Downloads/"pem_user_privatekey_download (1).pem" ~/Downloads/zdep-key.pem

Or simply rename it in Finder (macOS) or your file manager.

Then, for MacOS or Linux, you'll need to set the correct permissions:

chmod 600 ~/Downloads/zdep-key.pem

#2.2 SSH to the z/OS Instance

MacOS / Linux

ssh zosadmn@<ip-address> -i ~/Downloads/zdep-key.pem

Windows (PowerShell)

ssh zosadmn@<ip-address> -i C:\Users\YourName\Downloads\zdep-key.pem

Replace <ip-address> with the z/OS IP address from your TechZone reservation.

Upon initial login, you may be prompted to accept the host key — type yes and hit Enter.

#2.3 Set TSO Password

Run the password change script:

password_change.py -u ZOSADMN -p <new_passphrase>

RACF Passphrase Rules:

  • Minimum 14 characters, maximum 100
  • At least 2 alphabetic characters (A–Z, a–z)
  • At least 2 non-alphabetic characters (numbers, punctuation, or spaces)
  • No more than 2 consecutive identical characters
  • Cannot contain your user ID

#Step 3: Connect via 3270 Terminal

#3.1 Open Your Terminal Emulator

Open your 3270 terminal emulator (e.g., IBM Host On-Demand).

#3.2 Configure the Connection

Create a new connection with these settings:

  • Host: The z/OS IP address from your TechZone reservation
  • Port: 2023
  • Terminal Type: IBM-3278-2

#3.3 Log In to TSO

Once connected, log in:

TSO ZOSADMN

Enter the passphrase set in Step 2 when prompted.

#Troubleshooting

#Cannot Reach the z/OS IP Address

  • Confirm VPN is connected
  • Verify the IP address from your TechZone reservation details
  • Check that the instance is running (not stopped or expired)

#TSO Login Fails

  • Verify you are using username ZOSADMN
  • Confirm the passphrase was set correctly in Step 2
  • Re-run the password change script if needed

#SSH Permission Denied

  • Check the key file permissions: chmod 600 your-key.pem
  • Verify you are using the correct key file from your TechZone reservation
  • Ensure VPN is connected before attempting SSH
