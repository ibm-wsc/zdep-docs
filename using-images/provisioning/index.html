#Provisioning z/OS

Learn how to provision z/OS demo images on IBM TechZone.

⚠️ Note: Cisco AnyConnect is required to access the on-premise z/OS environment provisioned with this collection. If you do not want to, or cannot, install it on your computer, you'll need to provision a Jump Server (a Windows VM that comes with Cisco installed) to connect through.

#Overview

IBM TechZone provides on-demand access to pre-configured z/OS demo environments. Each image is fully configured and ready to use for demonstrations and hands-on labs.

#Provisioning Steps

#1. Choose an Image

Not sure which image to use? Browse the Image Catalog to see what's available and pick the right one for your demo.

#2. Access TechZone

Navigate to IBM TechZone and find the Collection named "z/OS Customized Demo Images".

#3. Select Your Image

On the left-hand navigation menu, select "z/OS Demo Catalog" and browse the available images and Collections.

Note: Tiles here are either blue, meaning it is just the environment (usually a more simple image, a base configuration of CICS, for example), or purple, meaning it's a link to a Collection, featuring a configured demo/lab and its corresponding documentation (for example, Machine Learning for z/OS). All images available in this list used this program to develop the customized image, built from the z/OS Standard Image.

#4. Submit a Reservation

  • To start a reservation, hover over the button on the desired image that reads "systems-2 environment" and click on the blue button that changes to Reserve It.
  • It will then open a new tab. Click on the checkbox next to Request an environment:
  • This will then open a form for you to fill out with reservation details. Give it a relevant Name for your purposes. Then select a Purpose:

The four options are Demo, Pilot, Education, and Test.

Each Purpose has its own Duration Policy. To the view the policy for each, click on one and then use the right-hand drop-down menus to select Duration Policy.

  • The two longer-lasting purposes, Demo and Pilot, require a valid IBM Sales Cloud Opportunity ID. This is to ensure that longer running instances are aligned with TechZone's intended purpose - for technical pre-sales and IBM business partners to provide clients with demonstrations and labs. Just the ID, no URL. It will be validated (has to be real, cannot be 'Lost' or 'Won' already)
  • The two shorter purposes, Education and Test, only require a Purpose description.
  • Select a Geography from the dropdown list. At the moment, only North America is available.
  • After selecting Geography, additional fields will appear. Next, select the desired date/time range to begin and end the reservation.

Remember to allow for ~1 hour for provisioning + ample time to prepare for the demo, ideally taking a video fo the demo ahead of time if possible.

  • If using the z/OS Standard Image, it will prompt you for vCPU and Memory:

Other images have preset values for these corresponding to the needs of that specific demo.

  • Next, click on the Check Availability button.
  • After a moment, if there is available capacity a success message will pop up saying.
  • Next, click on the checkbox to agree to the terms, and then click the Submit button.
  • It will take you to a page that says "Thank you" with a button to see "My Reservations".
  • Click on the "My Reservations" button there, or use the "My TechZone" dropdown at the top of the page to go to "My Reservations" later to see your active and pending reservations.
  • It will first show as Scheduled in grey for about 5 minutes, and then switch to Provisioning in yellow.
  • When the environment is ready (usually ~1 hour), you will receive an email from noreply@techzone.ibm.com with subject "Your environment is ready" that contains your instance's Reservation ID and the reservation will change to green and show Status - Ready.
  • If the environment fails to provision, you will receive an email from noreply@techzone.ibm.com with subject "An action related to your environment has failed". To see the details of a failed reservation, use the Filter by Status dropdown and select statuses Failed, Deleted or Expired.

#What You'll Receive

Once 'Ready', the following information is provided:

  • z/OS IP address
  • Linux host IP address
  • VPN connection details (if provisioned by an external user)
  • An SSH private key
