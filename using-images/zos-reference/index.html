#TSO/ISPF

#TSO Logon

First time? If you haven't set your TSO password yet, see Connecting to Your Instance before proceeding.

After connecting via 3270 emulator, log in with:

TSO ZOSADMN

And use the password you set previously via SSH.

Haven't set your TSO password yet? Go here.

After login, the ISPF Primary Option Menu appears:

------------------------- ISPF Primary Option Menu --------------------------
Option ===>

   0  Settings          Terminal and user parameters
   1  View              Display source data or listings
   2  Edit              Create or change source data
   3  Utilities         Perform utility functions
   4  Foreground        Interactive language processing
   5  Batch             Submit job for language processing
   6  Command           Enter TSO or Workstation commands
   7  Dialog Test       Perform dialog testing
   9  IBM Products      IBM program development products
  10  SDSF              System Display and Search Facility

Enter X to Terminate using log/list defaults

#Viewing Job Output

  1. Enter O to see output queue
  2. Enter S next to the job name
  3. Select dataset to view: JESMSGLG, JESJCL, JESYSMSG, or step output

# Linux/Mac
ssh zosadmn@<ip-address>

#File Permissions

ls -l                  # View permissions
chmod 755 script.sh    # rwxr-xr-x
chmod 644 file.txt     # rw-r--r--
chmod +x script.sh     # Add execute
chown user:group file  # Change ownership

#Accessing MVS Datasets from USS

# Copy MVS dataset to USS file

# View a PDS member
cat "//'USER.TEST.CNTL(MEMBER)'"

# Run TSO commands from USS
tsocmd "LISTDS 'USER.*.CNTL'"

#Dataset Reference

#Naming Conventions

HIGH.LEVEL.QUALIFIER.LOW.LEVEL.QUALIFIER

Rules:

  • Max 44 characters total, each qualifier max 8 characters
  • Start with alphabetic or national character (@, #, $)
  • Qualifiers separated by periods

Examples:

USER.TEST.CNTL          /* User test control library */
SYS1.PARMLIB            /* System parameter library */
USER.PROD.COBOL.SOURCE  /* Production COBOL source */

#Dataset Types

TypeDescriptionExample
PSSequential — single stream of recordsUSER.TEST.DATA
PDSPartitioned — library with named membersUSER.TEST.CNTL(MEMBER1)
VSAMKey-sequenced or entry-sequencedUSER.TEST.VSAM.CLUSTER
HFS/zFSUnix hierarchical filesystem/u/userid/file.txt

/* ISPF — Option 3.4 */

#Allocating Datasets

/* TSO */
ALLOC DA('USER.NEW.DATA') NEW SPACE(5,5) TRACKS +
  DSORG(PS) RECFM(F,B) LRECL(80) BLKSIZE(27920)

/* PDS/library */
ALLOC DA('USER.NEW.CNTL') NEW SPACE(5,5) TRACKS +
  DIR(10) DSORG(PO) RECFM(F,B) LRECL(80) BLKSIZE(27920)

#Dataset Attributes

#Troubleshooting

#Cannot Log On to TSO

  • Verify the passphrase was set correctly (see Connecting to Your Instance)
  • Username is ZOSADMN (with an N)

#Cannot Access Dataset

LISTCAT ENTRIES('DATASET.NAME')  /* Check if it exists */
LISTDS 'DATASET.NAME' STATUS     /* Check status */
HRECALL 'DATASET.NAME'           /* Recall if migrated */

#USS Permission Denied

ls -l file         # Check permissions
chmod 755 file     # Fix permissions
chown user file    # Fix ownership

#Job Fails

  1. Check JESMSGLG for system messages
  2. Check JESJCL for JCL errors
  3. Check JESYSMSG for allocation errors
  4. Check step output for program errors

#Slow Response

  • Check system load in SDSF (DA panel)
  • Try during off-peak hours
  • Use batch for large jobs
