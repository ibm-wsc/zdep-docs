#CICS Troubleshooting

#Region Won't Start

Check Liberty Angel is running:

/D A,BBGZANGL

If not active, start it: /S BBGZANGL

Check CICS SVC is installed:

/D PROG,LPA,CSECT=DFHCSVC

Check log streams are defined:

/D LOGGER,LOGSTREAM,LSN=CICSPRD.CICS62.*

View region message log:

tail -100 /var/cicsts/regions/CICS62/logs/MSGUSR

#CMCI Connection Fails

Verify CMCI is listening on port 10080:

ssh zosadmn@<zos-ip>
netstat -an | grep 10080

Test CMCI manually:

curl -u CICSUSER:<password> \
  http://<zos-ip>:10080/CICSSystemManagement/CICSRegion/CICS62

If the region is not running, check MSGUSR for startup errors.

#Program Not Found (DFHPI1008)

Verify the program is in the load library:

tsocmd "LISTDS 'CICS62.CICS.LOADLIB' MEMBERS"

Verify the program is defined in CICS:

curl -u CICSUSER:<password> \
  "http://<zos-ip>:10080/CICSSystemManagement/CICSDefinitionProgram/CICS62?CRITERIA=(NAME=HELLO)"

If not defined, define it via CMCI — see CICS Overview for configuration details.

#Transaction Abend

Check the transaction is defined: In CICS, enter: CEDA DI TR(HELO)

View abend details in SDSF:

/F SDSF
ST

Find the CICS job and check CSMT or MSGUSR output.

#Bundle Won't Install

Check the bundle directory exists:

ls /var/cicsts/cics_bundles/

Check CMCI bundle definition:

curl -u CICSUSER:<password> \
  "http://<zos-ip>:10080/CICSSystemManagement/CICSDefinitionBundle/CICS62?CRITERIA=(NAME=TERMINAL)"

Check bundle name length — CICS resource names are limited to 8 characters. Bundles are truncated automatically; confirm the name in the CMCI output matches what you expect.

Bundle already exists — the Ansible playbook automatically performs a disable/enable cycle to refresh existing bundles. If deploying manually, disable then re-enable the bundle definition in CICS.

Java errors — verify java_home in vars.yml and confirm Java is accessible on z/OS. Check jar command permissions.

#Transaction Not Found After Bundle Install

Verify the bundle is installed:

curl -u CICSUSER:<password> \
  "http://<zos-ip>:10080/CICSSystemManagement/CICSDefinitionBundle/CICS62?CRITERIA=(NAME=TERMINAL)"

Check bundle is enabled — in CICS enter:

CEDA DI BU(TERMINAL)

Verify transactions are defined in cics.xml inside the bundle's META-INF/ directory.

#Compilation Failure

Common causes:

  • Missing Language Environment datasets (SYS1.CEE.*)
  • Incorrect permissions on CICS62.CICS.SOURCE or CICS62.CICS.LOADLIB
  • Syntax errors in the source

Review the SYSPRINT output in SDSF for the specific compiler error.
