#CICS Java Bundles

CICS bundles are packaged applications deployed to CICS regions via CMCI. The z/OS Standard + CICS image ships with two bundles pre-deployed and supports adding more using CICS Explorer, Maven, or Gradle.

#Deployed Bundles

#terminal — Terminal I/O Examples

Demonstrates 3270 terminal I/O operations using the JCICS API.

TransactionDescription
JTM1Terminal I/O example

Usage: At a 3270 terminal, enter the transaction followed by optional arguments:

JTM1 Hello from CICS!

Output:

JTM1 Hello from CICS! - Starting TerminalExample1
Arg 0 : JTM1
Arg 1 : HELLO
Arg 2 : FROM
Arg 3 : CICS!
Completed TerminalExample1

Note: Output assumes upper-casing is enabled for the terminal.

#vsam — VSAM File Examples

Demonstrates VSAM file operations (KSDS, ESDS, RRDS) using the JCICS API.

TransactionDescription
JVK1–JVK5VSAM KSDS (Key-Sequenced Data Set) examples
JVE1–JVE5VSAM ESDS (Entry-Sequenced Data Set) examples
JVR1–JVR5VSAM RRDS (Relative Record Data Set) examples

Usage:

JVK1

Output:

JVK1 - Starting KsdsExample1
VSAM KSDS record addition example
Wrote record with key 0x0003E712
Completed KsdsExample1

#Bundle Naming Requirements

CICS resource names are limited to 8 characters. Bundle names are automatically truncated:

  • terminal.zip → TERMINAL ✓
  • vsam.zip → VSAM ✓
  • longnameapp.zip → LONGNAME (truncated)

Use short, descriptive names when adding bundles.

#Bundle Layout on z/OS

Bundles are deployed to /var/cicsts/cics_bundles/, each in its own subdirectory:

/var/cicsts/cics_bundles/
├── terminal/
│   ├── META-INF/cics.xml
│   └── [bundle contents]
└── vsam/
    ├── META-INF/cics.xml
    └── [bundle contents]

To refresh an existing bundle, disable then re-enable it in CICS.

#Deployment Methods

#CICS Explorer (Eclipse)

  1. Install CICS Explorer in Eclipse
  2. Configure a CMCI connection — host, port 10080, user CICSUSER
  3. Import or create a CICS bundle project
  4. Right-click → Export Bundle Project to z/OS UNIX File System
  5. Define and install the bundle from the CICS Explorer views

#Maven Plugin

<plugin>
    <groupId>com.ibm.cics</groupId>
    <artifactId>cics-bundle-maven-plugin</artifactId>
    <version>1.0.3</version>
    <executions>
        <execution>
            <goals><goal>deploy</goal></goals>
            <configuration>
                <url>https://<zos-ip>:10080</url>
                <username>${cmci.username}</username>
                <password>${cmci.password}</password>
                <bunddef>MYBUNDLE</bunddef>
                <csdgroup>MYGROUP</csdgroup>
                <region>CICS62</region>
            </configuration>
        </execution>
    </executions>
</plugin>

mvn com.ibm.cics:cics-bundle-maven-plugin:deploy

#Gradle Plugin

plugins {
    id 'com.ibm.cics.bundle' version '1.0.3'
}

deployCICSBundle {
    url = 'https://<zos-ip>:10080'
    username = project.findProperty('cmci.username')
    password = project.findProperty('cmci.password')
    bunddef = 'MYBUNDLE'
    csdgroup = 'MYGROUP'
    region = 'CICS62'
}

./gradlew deployCICSBundle

Getting CMCI credentials: Your CMCI username is CICSUSER. The password is the CICSUSER RACF password set on your instance — see Configuration Details for connection details.

#Creating Your Own Bundles

#Bundle Structure

mybundle/
├── META-INF/
│   └── cics.xml
└── myapp.jar

#Example cics.xml

<?xml version="1.0" encoding="UTF-8"?>
<manifest xmlns="http://www.ibm.com/xmlns/prod/cics/bundle"
          bundleMajorVer="1" bundleMinorVer="0" bundleMicroVer="0"
          bundleRelease="0" bundleVersion="1" id="mybundle">
  <define name="MYAPP" path="myapp.jar"
          type="http://www.ibm.com/xmlns/prod/cics/bundle/OSGIBUNDLE"/>
</manifest>

#Packaging

cd mybundle
jar -cvf ../mybundle.zip *

#Sample Bundles Repository

The terminal and vsam bundles are based on IBM's CICS Java samples:

GitHub: https://github.com/siler23/cics-java-jcics-samples

Other available samples:

SampleDescription
linkEXEC CICS LINK to call other programs
serializeSerialization techniques
tdqTransient Data Queue (TDQ) operations
tsqTemporary Storage Queue (TSQ) operations
terminal3270 terminal I/O (deployed by default)
vsamVSAM file operations (deployed by default)

To build a sample and deploy it using the Maven plugin:

git clone https://github.com/siler23/cics-java-jcics-samples.git
cd cics-java-jcics-samples/projects/com.ibm.cicsdev.link
mvn clean verify

Then deploy the resulting .zip using CICS Explorer or the Maven plugin above.
