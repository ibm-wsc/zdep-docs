#z/OS Standard Image System Configuration

All demo images are built on a common base z/OS system called the z/OS Standard Image. This page describes the standard DASD layout, storage configuration, and installed software that is present in every image.

#DASD Volume Configuration

The z/OS system is configured with 3390 DASD volumes. To display all online volumes from SDSF, enter:

/D U,DASD,ONLINE,,256
Volume SerialDescription
LTxRS1z/OS system datasets including the Master Catalog
LTECATCritical system configuration datasets (Master Catalog, IODF, ACDS/SCDS, MAN datasets, LOGREC)
PAGE01Page datasets for virtual memory management
JESPL1JES2 spool and checkpoint datasets
USER01Non-SMS managed user space
USMS01SMS-managed user space
DUMP01Dynamic dump allocation

Note: USER01 and USMS01 persist across Recreate operations (when using the same Instance Name). All other volumes are re-created from the image.

#System Managed Storage (SMS)

SMS is active on the system. Control datasets are located on LTECAT:

DatasetDescription
LTEPLEX.SMS.ACDSSMS active control dataset
LTEPLEX.SMS.COMMDSSMS common dataset
LTEPLEX.SMS.SCDSSMS system control dataset

SMS constructs are assigned automatically based on dataset naming patterns:

Storage ClassStorage GroupUsed For
STANDARDSTANDARDGeneral SMS-managed datasets
DUMPDSDUMPDSSystem dump files (SYS1.DUMP.**)

Logstream datasets (LOGGER.**, DASDLOGR.**, IXGLOGR.**) are assigned the MVSLOGR data class and STANDARD storage class.

The base image includes z/OS 3.1 with the following software. Additional middleware (CICS, Db2, IMS, MQ) is installed on top of this base depending on the image selected.
