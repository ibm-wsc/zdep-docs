#SMS Routine Summary

The SMS configuration uses automated routines to assign SMS constructs based on dataset characteristics. These routines ensure consistent storage management across the system:

Key Features:

  • Automatic Assignment: SMS constructs are automatically assigned based on dataset naming patterns and user specifications
  • Logstream Management: Special handling for system logstreams (LOGGER., DASDLOGR., IXGLOGR.**) with MVSLOGR data class and STANDARD storage class/group
  • Dump Dataset Management: Dedicated storage for system dumps (SYS1.DUMP.**) using EXTENDED data class and DUMPDS storage class/group
  • User Control: User-specified data classes and storage classes are honored when provided
  • Fail-Safe: Undefined storage class/group combinations result in allocation errors to prevent misconfiguration

Assignment Flow:

  1. Storage Class is evaluated first based on dataset name patterns
  2. Data Class is assigned based on dataset patterns (independent of storage class)
  3. Management Class is assigned based on the storage class value
  4. Storage Group is assigned based on the storage class value
