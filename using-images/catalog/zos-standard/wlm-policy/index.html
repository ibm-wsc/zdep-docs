#WLM Policy Default

#Overview

This document defines the default Workload Manager (WLM) policy configuration for the z/OS Standard Image. The WSCDEF policy serves as the baseline starter policy and provides a comprehensive framework for workload management across all major subsystem types.

Policy Information:

  • Name: WSCDEF
  • Description: WSC WLM starter policy
  • Level: 038
  • Product: WLM AA zOS V2R4 HBB77C0

#Resource Management Strategy

#Resource Groups

#1. RGCAPPED - Severe Resource Limitation

Purpose: Control quiesced nonswappable workloads that need to be severely restricted

Configuration:

  • Type: CPU Service Units
  • Capacity Minimum: 0
  • Capacity Maximum: 1
  • Include Specialty Processors: Yes

Use Cases:

  • Problematic workloads requiring containment
  • Testing resource limitation scenarios
  • Temporary workload restriction during incidents

#2. RGCPCAP - General Purpose CPU Capping

Purpose: Cap General Purpose CPU consumption without affecting specialty engines

Configuration:

  • Type: CPU Service Units
  • Capacity Maximum: 5000
  • Include Specialty Processors: No

Use Cases:

  • Low priority batch workloads
  • Background processing
  • Non-critical batch jobs

#3. RGZCX - z/OS Container Extension Tenant Management

Purpose: Enforce IBM License Metric Tool (ILMT) compliance for container workloads

Configuration:

  • Type: Number of CPs × 100
  • Capacity Maximum: 170 (equivalent to 1.7 CPs)
  • Include Specialty Processors: Yes
  • Tenant ID: DEPT1234
  • Solution ID: D9F4555-13446AD-70FF8DA303-0915801A-C0AA-489A-891B-6EF7DD-C744F4

Use Cases:

  • z/OS Container Extension instances
  • Department-level resource allocation
  • License compliance enforcement

#Workload Classification Framework

#1. Batch Workloads (BAT_WKL)

#Service Classes

#BATHI - High Priority Batch

Goal: Velocity-based with dual importance levels

  • Period 1: Importance 3, Duration 250ms, Velocity 30%
  • Period 2: Importance 4, Velocity 20%

Characteristics:

  • CPU Critical: No
  • I/O Priority: Normal
  • Honor Priority: Default

Target Workloads:

  • Production batch jobs requiring faster completion
  • Time-sensitive batch processing
  • Critical overnight batch windows
#BATLO - Low Priority Batch

Goal: Velocity-based with resource capping

  • Importance: 5
  • Velocity: 10%
  • Resource Group: RGCPCAP (5000 service units cap)

Characteristics:

  • CPU Critical: No
  • I/O Priority: Normal
  • Honor Priority: Default

Target Workloads:

  • Background batch processing
  • Non-critical reports
  • Maintenance jobs
#BATTEST - Test Batch

Goal: Discretionary (lowest priority)

Characteristics:

  • CPU Critical: No
  • I/O Priority: Normal
  • Honor Priority: Default

Target Workloads:

  • Development batch jobs
  • Testing and QA batch processes
  • Non-production batch work

#Classification Rules

#Subsystem Type Coverage

The standard policy includes classification rules for the following subsystem types:

  1. ASCH - APPC scheduled transactions
  2. CB - WebSphere Application Server (Liberty)
  3. CICS - CICS transaction processing
  4. DDF - Distributed Data Facility (DB2)
  5. IMS - IMS transaction processing
  6. IWEB - Scalable WebServer
  7. JES - Job Entry Subsystem
  8. LDAP - LDAP enclave
  9. LSFM - LAN Server for MVS
  10. MQ - MQ Series Workflow
  11. OMVS - Unix System Services
  12. SOM - SOM client objects
  13. STC - Started Tasks
  14. TCP - TCP independent enclave
  15. TSO - Time Sharing Option

#Detailed Classification Rules by Subsystem

#JES (Batch) Classification

Low Priority Classes:
- Class J → BATLO (RCBATTCJ)
- Class E → BATLO (RCBATTCE)
- Class M → BATLO (RCBATTCM)

Production Classes (via PBAT_TCG):
- Classes A, B, C → BATHI (RCPBAT)

Test Classes (via TBAT_TCG):
- Classes X, Y, Z → BATTEST (RCTBAT)

#CICS Classification

Region-Specific (CICSIP* subsystem):
- TRN1 → CICSHI (RCTRN1)
- TRN2 → CICSHI (RCTRN2)
- TRN3 → CICSHI (RCTRN3) [Mobile attribute]

Global Transaction Names:
- TRX1 → CICSHI (RCTRX1)
- TRX2 → CICSLO (RCTRX2)

Default: CICSLO (RCUCICS)

#DDF (DB2) Classification

DB2P* Subsystems (Production):
- Stored Procedure CALLSPA → DDFHI (RCSPAS1)
- Userid SYSADM → DDFHI (RCSYSADM)
- Plan ACCTRECV → DDFHI (RCACCTRC)
- Default → DDFLO (RCDDFDBC)

DB2T* Subsystems (Test):
- Stored Procedure PAYABLE → DDFLO (RCPAYT)
- LU Name LU12345% → DDFLO (RCLU1234)
- Default → DDFLO (RCDB2T)

Default: NEWWORK (RCUDDF)

#IMS Classification

IMSIP* Subsystems:
- Transaction IVTNO* → IMSHI (RCIVTNO)
- Default → IMSHI (RCIMSIP)

Default: IMSHI (RCUIMS)

#OMVS Classification

Userid OMVSKERN → OMVS (RCOMVSKN)
Transaction FTP* → STCHI (RCFTP)
Transaction INET* → STCHI (RCINET)

Default: NEWWORK (RCUOMVS)

#Started Task (STC) Classification

Critical Infrastructure:
- DB2*IRLM → SYSSTC (RCDB2IRL)
- MQ%%MSTR → STCCRIT (RCMQMSTR) [Boosted]
- MQ%%CHIN → STCHI (RCMQCHIN) [Boosted]
- WAS* → STCHI (RCWASSTC) [Boosted]

Database Systems:
- PDBL_TNG → STCCRIT (RCPDBSTC) [Boosted]
- TDBL_TNG → STCLO (RCTDBSTC) [Region Goal]
- %%%%SPAS → STCHI (RCDBSPAS)

Online Systems:
- PONL_TNG → STCHI (RCPRODON) [Boosted, Region Goal]
- TONL_TNG → STCLO (RCTONL) [Region Goal]
- CICTOR* → SERVERS (RCTOR) [Boosted, Region Goal Both]

Server Regions:
- SRVR_TNG → SERVERS (RCSRVR) [Boosted]

Container Extension:
- ZCX* → ZCXHI (RCZCX)

System Tasks:
- SYST_TNG → SYSSTC (RCSYSSTC)
- SYHI_TNG → SYSSTC (RCMON)
- SYLO_TNG → STCLO (RCSYSLO)
- STOR_TNG → STCLO (RCSTOR)
- ZOSC_TNG → STCHI (RCZOSCON)

SPM Rules:
- SubsystemParameter SYSTEM → SYSTEM (RCSPMSTM)
- SubsystemParameter SYSSTC → SYSSTC (RCSPMSTC)

Default: NEWWORK (RCUSTC)

#TSO Classification

EMG_UIG (Emergency Userids) → SYSSTC (RCTSOEMG)
TSO_UIG (General Userids) → TSO (RCTSO)

Default: NEWWORK (RCUTSO)

#WebSphere (CB) Classification

Collection Name Based:
- BBTRD2* → WASHI (RCWASTRD)
- BBCICS* → WASHI (RCWASCIC)
- BBIVP* → WASLO (RCWASIVP)
- BBERWW* → WASLO (RCWASBER)
  - Transaction Class TCITPI → WASLO (RCWASTCI)

Liberty Servers:

  - Transaction Class TCADMIN → WASLO (RCWASADM)
- ZCEE* → WASHI (RCWASZCE)
  - Transaction Class TCCICS → WASHI (RCWASCIC)
  - Transaction Class TCMQ → WASHI (RCWASMQ)
  - Transaction Class TCIC → WASLO (RCWASIC)

Default: NEWWORK (RCUWAS)

#Report Classes

The standard policy includes 68 report classes for comprehensive workload tracking and chargeback. Report classes are organized by:

#Application Categories

  • CICS transactions (RCCICSIP, RCTRN1-3, RCTRX1-2, RCTOR)
  • IMS transactions (RCIMSIP, RCIVTNO)
  • DB2/DDF work (RCDDFDBC, RCDB2T, RCDB2IRL, RCDBSPAS, RCSPAS1)
  • WebSphere (RCWAS* series - 15 classes)
  • MQ Series (RCMQMSTR, RCMQCHIN)

#Infrastructure Categories

  • Batch work (RCBATTCE, RCBATTCJ, RCBATTCM, RCPBAT, RCTBAT)
  • System tasks (RCSYSSTC, RCMON, RCSYSLO)
  • Storage (RCSTOR)
  • Network (RCFTP, RCINET, RCLDAP, RCTCP)
  • z/OS Connect (RCZOSCON)
  • z/OS Container Extension (RCZCX)

#Organizational Categories

  • Production work (RCPRODON, RCPDBSTC, RCPBAT)
  • Test work (RCTONL, RCTDBSTC, RCTBAT)
  • Unclassified work (RCUASCH, RCUCICS, RCUDDF, RCUIMS, etc.)

#Special Purpose

  • Account receivables (RCACCTRC)
  • Payroll (RCPAYT)
  • System administration (RCSYSADM)
  • SPM rules (RCSPMSTM, RCSPMSTC)

#Service Configuration

#Service Coefficients

CPU:  1.0  (CPU time fully counted)
IOC:  0.0  (I/O operations not counted)
MSO:  0.0000 (Memory not counted)
SRB:  1.0  (SRB time fully counted)

Rationale: This configuration focuses WLM management on CPU consumption, which is typically the most constrained resource in modern z/OS environments.

#Service Options

I/O Priority Management: Disabled
Dynamic Alias Management: Disabled
I/O Priority Groups: Disabled
Discretionary Goal Management: Active

#Service Policy

#PROD - Production Policy

Description: Default policy
Overrides: None defined

This policy serves as the baseline with no service class, resource group, or tenant resource group overrides. All workload management is handled through the base definitions.
