#z/OS Standard + Linux Sidecar

The z/OS Standard + Linux Sidecar image extends the z/OS Standard Image with a co-located Linux virtual machine. The Linux host acts as a network gateway for z/OS, forwarding ports through to z/OS over a tunnel network. A separate sidecar Linux VM lives in the same VLAN and can reach z/OS directly — useful for running automation, tooling, or integration workloads alongside z/OS.

#What's Included

  • z/OS 3.1 — Full z/OS Standard Image — see z/OS Standard Image for the complete software list
  • Linux gateway host — Sits between the public IP and z/OS; all inbound connections to z/OS pass through it via port forwarding
  • Linux sidecar — A separate Linux VM in the same VLAN with direct network access to z/OS

#Network Architecture

In this image, z/OS does not have a directly reachable public IP address. Instead, the underlying Linux gateway host owns the reservation IP and forwards connections through to z/OS over a tunnel network. Both the Linux gateway and z/OS share the same externally visible IP address.

#Port Forwarding Reference

All connections to z/OS are made to the reservation IP on the forwarded port. The Linux gateway transparently forwards to z/OS.

Note: SSH to z/OS uses port 2222 (not the standard 22). Port 22 on the reservation IP connects to the Linux gateway itself.

#Connecting to the Linux Sidecar

The sidecar Linux VM is accessible separately — its connection details are provided alongside the z/OS reservation details in TechZone. It has direct network access to z/OS on the tunnel network and can be used to run scripts, automation pipelines, or integration tooling without going through the port-forwarding layer.

#Use Cases

  • Demonstrations that require a Linux host running alongside z/OS (e.g. Ansible, containerised tooling, REST clients)
  • Integration scenarios where a Linux workload calls z/OS services
  • Environments where you want to isolate the z/OS network behind a Linux jump/proxy
