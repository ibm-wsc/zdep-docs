#Connecting to z/OS from the Jump Server

🏗️ Page Under Construction - content coming soon

Once your provisioned jump server reservation is Ready, open it and use the pre-installed tools to connect to your z/OS instance, including:

  • Cisco AnyConnect
  • IBM Personal Communications (PCOMM)
  • VS Code
  • GitHub Desktop
  • Firefox

⚠️ Note: IBMers only need their regular Cisco AnyConnect to access the environment. External users need to download AnyConnect or connect through the jump server.

#1. Open the Jump Server

  • Find your jump server in My Requests and click the Open this reservation button on its tile.
  • This opens a new tab. Click on the twisty dropdown next to your request to show the connection details.

#2. SSH to z/OS (to set your TSO password)

Before connecting via 3270 terminal, you need to set your TSO password via SSH. This is a one-time step after provisioning.

#2.1 Get your SSH key into the jump server

Your TechZone SSH key needs to be accessible from inside the jump server. The easiest way is to open a browser inside the jump server session, navigate to your TechZone reservation, and download the SSH key directly from there.

#2.2 SSH to z/OS

Open PowerShell or Command Prompt from the Start menu and run:

ssh zosadmn@<zos-ip-address> -i C:\Users\<username>\Downloads\zdep-key.pem

Replace <zos-ip-address> with the z/OS IP from your z/OS TechZone reservation (not the jump server).

Upon first connection you may be prompted to accept the host key — type yes and press Enter.

#2.3 Set TSO password

password_change.py -u ZOSADMN -p <new_passphrase>

RACF Passphrase Rules:

  • Minimum 14 characters, maximum 100
  • At least 2 alphabetic characters (A–Z, a–z)
  • At least 2 non-alphabetic characters (numbers, punctuation, or spaces)
  • No more than 2 consecutive identical characters
  • Cannot contain your user ID

#3. Connect via 3270 Terminal

IBM Host On-Demand is pre-installed on the jump server desktop.

  1. Open IBM Host On-Demand from the desktop or Start menu
  2. Create a new session with the following settings:
SettingValue
Hostz/OS IP address from your TechZone reservation
Port2023
Terminal TypeIBM-3278-2
  1. Click Connect
  2. At the z/OS login prompt, type TSO ZOSADMN and press Enter
  3. Enter the passphrase set in Step 2 when prompted

#Troubleshooting

#Cannot reach the z/OS IP from the jump server

  • Confirm both the jump server and z/OS reservations are in Ready state
  • Use the z/OS IP from the z/OS reservation details — not from the jump server details
  • Both reservations should be reachable over the IBM network without additional VPN

#Browser session disconnects

  • The jump server session times out after a period of inactivity
  • Re-open the environment from your TechZone reservation — session state is preserved as long as the reservation is active
